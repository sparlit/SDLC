# /documentation-update

Use this workflow when working on **documentation-update** in `SDLC`.

## Goal

Updates or overhauls project documentation, including README and guides.

## Common Files

- `README.md`

## Suggested Sequence

1. Understand the current state and failure mode before editing.
2. Make the smallest coherent change that satisfies the workflow goal.
3. Run the most relevant verification for touched files.
4. Summarize what changed and what still needs review.

## Typical Commit Signals

- Edit README.md to add or update documentation, specifications, or guides.
- Review and finalize documentation changes.
- Commit changes with a message indicating documentation update or overhaul.

## Notes

- Treat this as a scaffold, not a hard-coded script.
- Update the command if the workflow evolves materially.
